《專案簡述》
　－－
　此為交易平台之中，「買賣訂單配對執行」的簡易實作。僅依賴 C++20 標準庫，具有粒度較大的執行緒安全。
　此案主要用於驗證一些設計思路，代碼從簡實作，避免主題失焦。後續若要擴展，可以著手於「更細膩的執行緒安全」「更進階的訂單下單邏輯（Ex：IOC、FOK）」。
　－－
　此中主要設計思路之一（1/2），乃是「Std 之下，使用智慧指標作為元素的，容器物件」。此中容器物件，具有能夠主動控制調用時機的 GC（Garbage Collection）功能。
　容器物件的元素移除和元素釋放，能於作業流程之下平移到適當時機集中執行。容器物件本身，因此能於服務繁忙時期，維持較好的執行效能。
　－－
　此中主要設計思路之二（2/2），乃是「買賣訂單間的交易互動，不是直接數值增減，而是媒介矢量增減」。此法靈感源自 TaifexMp 下的 TcpMessage 中的資料結構。
　買賣訂單之間若是構成交易（將要發生數值增減），會先構成一個交易請求（矢量、SubOrderMatchRequest），再以該個交易請求執行交易（實際執行數值增減）。此中「交易請求」，用於收斂關於交易操作的所有控制，包括超量交易防範、進階下單邏輯實現、異常交易攔截、⋯⋯。
　買賣訂單之間如此互動，更加符合會計直覺，更易基於會計需求擴展維護事務邏輯（Ex：流水日誌、交易存根、資金流動軌跡、⋯⋯）。
